    物理信息强化学习 (PIRL)¶

    在不增加仿真复杂度的前提下，让智能体"理解"物理

    为什么需要 PIRL¶

    当前 RL 的局限¶

    标准 DRL（包括本项目的 ID3QN）将环境当作黑盒：智能体只看到 LiDAR 距离和目标方位，对无人艇的物理特性一无所知 -- 不知道自己有多重、惯性多大、转弯半径多少。

    当前 ID3QN 的视角:

  "前方 5 米有障碍物，目标在右前方 30°"        --> 选动作: 右转

问题:
  - 不知道当前速度 3 m/s 下转弯需要多大半径
  - 不知道急转会导致侧滑
  - 不知道高速状态下制动距离更长
  - 在仿真中"不需要知道"，但到了实船上差距就出来了

    仿真与实船的 Gap¶

    因素 c4nav-core 仿真 真实水面
    惯性 简化一阶模型 3-DOF + 附加质量
    转弯响应 即时 有延迟和过冲
    环境扰动 无 风、浪、流持续作用
    侧滑 无 高速转弯时显著

    核心问题：能否在不更换仿真器的前提下，缩小这个 Gap？

    PIRL 的思路¶

    不改变仿真器本身，而是通过观测和奖励让智能体主动感知物理量，训练出对物理特性更敏感的策略：

    传统 RL:
  环境 (黑盒物理) --> 几何观测 --> 智能体 --> 动作
                      "看到什么"

PIRL:
  环境 (同样的物理) --> 几何观测 + 物理量 --> 智能体 --> 动作
                        "看到什么 + 感受到什么"
                                     +
                        物理感知奖励: "急转弯是不好的"

    四个注入位置¶

    物理信息可以注入 RL 框架的 4 个位置：

                  +---------------------------+
              |      RL 训练循环           |
              |                           |
  +--------+  |  +---------+   观测 S     |
  | 环境   |--+--| 智能体  |<---------   |
  |        |  |  | (网络)  |--------->   |
  | A:转移 |<-+--| D:损失  |   动作 A    |
  | C:奖励 |  |  +---------+            |
  +--------+  |                           |
     B:观测   +---------------------------+

    逐位置详解¶

    位置 A：状态转移（动力学保真度）¶

    含义：提升仿真器本身的物理精度。

    USV 示例：将 c4nav-core 的一阶惯性模型替换为 Fossen 3-DOF 模型（包含附加质量、水动力阻尼、科里奥利力）。

    当前 (一阶惯性):
  v_next = v + α * (v_cmd - v)          # 指数趋近，无物理量纲

Fossen 3-DOF:
  M * v̇ + C(v)*v + D(v)*v = τ + τ_env  # 力/力矩平衡
  # M = 惯性矩阵 (含附加质量)
  # C = 科里奥利力 + 离心力
  # D = 水动力阻尼
  # τ_env = 风+浪+流 扰动力

    代价：需要依赖 MATLAB/Simulink 或重写动力学引擎，训练速度下降约 40 倍。

    本项目定位：属于第三阶段，通过对接 SpaceR-USV 实现，非第二阶段目标。

    位置 B：观测空间（智能体能看到什么）¶

    含义：在不改变物理模型的前提下，将动力学模型内部已有的物理量暴露给智能体。

    USV 示例：当前 c4nav-core 的动力学模型内部已经在计算速度和加速度（只是没有传给智能体），我们只需要把它们放入观测向量。

    # 当前观测 (184D):
obs = [lidar_180, dist_target, angle_target, dist_nearest_obs, angle_nearest_obs]
#       几何信息     导航目标        最近障碍物

# PIRL 观测 (190D):
obs = [lidar_180, dist_target, angle_target, dist_nearest_obs, angle_nearest_obs,
       v_surge,      # 纵向速度 (归一化)
       v_sway,       # 横向速度 (归一化) -- 用于感知侧滑
       yaw_rate,     # 转向率 (归一化)
       accel_surge,  # 纵向加速度 (归一化)
       accel_sway,   # 横向加速度 (归一化)
       accel_yaw]    # 角加速度 (归一化)

    为什么有用：

    物理量 智能体能学到什么
    v_surge (纵向速度) 高速时提前减速，低速时可以大胆转弯
    v_sway (横向速度) 感知侧滑，高速转弯时横向速度增大 = 危险
    yaw_rate (转向率) 感知当前转弯激烈程度
    accel_surge (纵向加速度) 感知加减速过程，学会更平滑的速度控制
    accel_yaw (角加速度) 感知转弯变化率，避免急转急停

    代码改动量：2 个文件，约 30 行。

    位置 C：奖励函数（鼓励什么行为）¶

    含义：在奖励中添加基于物理量的惩罚/奖励项。

    USV 示例：

    # envs/reward.py -- 新增 2 个物理感知奖励项

# 1. 能量消耗惩罚：频繁加减速 = 浪费燃料 + 不舒适
energy_penalty = -energy_scale * (abs(accel_surge) + abs(accel_sway))

# 2. 急转惩罚：角加速度过大 = 侧滑风险 + 机械损耗
jerk_penalty = -jerk_scale * abs(accel_yaw)

    USV 场景下的直觉：

    场景 无物理奖励 有物理奖励
    高速通过窄水道 可能急转避障，仿真中没问题 惩罚急转 -> 学会提前减速
    接近目标时 全速冲向目标，靠碰撞检测停止 惩罚大加速度 -> 学会平滑减速
    开阔水域 走折线也能到达 惩罚频繁变向 -> 学会走直线

    效果：训练出的策略在仿真中可能成功率略低（因为约束更严），但轨迹更平滑、更像真实船舶行为，迁移到实船时 Gap 更小。

    位置 D：网络/损失（物理方程作为约束）¶

    含义：在神经网络的损失函数中加入物理方程残差（类似 PINN）。

    为什么暂不采用：

    DQN 的训练目标是最小化 TD 误差：L = (Q(s,a) - y)²，其中 Q 值是"未来累计奖励的期望" -- 不是物理量，没有可以约束它的 PDE。

    PINN 方式 (适用于物理场求解):
  L = L_data + λ * L_physics
  L_physics = ||∂u/∂t + N[u]||²    # PDE 残差

DQN (不适用):
  Q(s,a) 不是物理场，不满足任何 PDE
  没有物理方程可以约束 Q 值

    可探索方向：辅助预测头（多任务学习），例如让网络同时预测"下一步速度"，用物理方程约束这个预测。但这是独立的研究贡献，不在第二阶段范围内。

    第二阶段方案：B + C¶

    选择位置 B 和 C 的理由：

    维度 A (改仿真) B (改观测) C (改奖励) D (改损失)
    代码改动 重写引擎 ~15 行 ~15 行 需要新研究
    训练速度影响 慢 40 倍 无影响 无影响 未知
    智能体代码改动 无 无 (state_dim 参数化) 无 需要新架构
    风险 高 极低 极低 高
    学术新颖性 低 (已有工作) 中 中 高

    实现细节¶

    B：观测空间扩展¶

    配置 维度 新增内容 配置文件
    基线（当前） 184 -- improved_d3qn.yaml
    + 速度 186 归一化纵向速度 + 转向率 pirl_v1.yaml
    + 速度 + 侧滑 188 + 归一化横向速度 + 角加速度 pirl_v2.yaml
    完整物理 190 + 纵向加速度 + 横向加速度 pirl_full.yaml

    代码改动位置：

    # usv_dynamics.py -- 在 get_state() 中暴露已有内部变量
def get_state(self):
    return {
        ...,
        'v_surge': self.v_surge / self.max_speed,         # 已有，只需归一化
        'v_sway': self.v_sway / self.max_speed,           # 已有
        'yaw_rate': self.yaw_rate / self.max_yaw_rate,    # 已有
        'accel_surge': self.accel_surge / self.max_accel,  # 需要缓存上一步
        'accel_sway': self.accel_sway / self.max_accel,    # 需要缓存上一步
        'accel_yaw': self.accel_yaw / self.max_yaw_accel,  # 需要缓存上一步
    }

# usv_env.py -- 在 _get_observation() 中拼接
def _get_observation(self):
    obs = [lidar, nav_features]
    if self.include_physics:  # YAML 开关
        obs.extend([v_surge, v_sway, yaw_rate,
                    accel_surge, accel_sway, accel_yaw])
    return np.array(obs, dtype=np.float32)

    智能体代码：零改动 -- state_dim 在 DQNAgent.__init__() 中已参数化，观测从 184 变成 190 时网络输入层自动适配。

    C：物理感知奖励¶

    # envs/reward.py -- 新增 2 项（配置开关控制）

if self.energy_scale > 0:
    energy = abs(accel_surge) + abs(accel_sway)
    reward -= self.energy_scale * energy       # 默认 0.1

if self.jerk_scale > 0:
    jerk = abs(accel_yaw)
    reward -= self.jerk_scale * jerk           # 默认 0.2

    YAML 配置新增：

    reward:
  # 现有塑形 (不变)
  enable_shaping: true
  progress_scale: 2.0
  safety_distance: 3.0
  heading_scale: 0.3
  # 新增: 物理感知奖励
  energy_scale: 0.1       # 能量消耗惩罚权重
  jerk_scale: 0.2         # 急转惩罚权重

    评估指标¶

    指标 含义 计算方式
    成功率 基本能力 success 列统计
    平均奖励 综合表现 reward 列均值
    收敛轮次 学习效率 首次连续 50 轮 SR > 70%
    轨迹平滑度 PIRL 核心 轨迹曲率方差（越小越平滑）
    能量消耗 PIRL 核心 累计 |accel|（越小越节能）
    急转次数 PIRL 核心 |accel_yaw| > 阈值的步数

    预期结论模式¶

    可能的结果:

情况 1 (理想):  A4 > A1，且 A4 轨迹更平滑
  -> 结论: PIRL (B+C) 在轻量仿真中有效

情况 2 (部分):  A4 成功率 ≈ A1，但轨迹显著更平滑
  -> 结论: PIRL 不改善导航成功率，但改善策略质量

情况 3 (负面):  A4 < A1
  -> 结论: 物理约束过强，限制了探索
  -> 后续: 调整 energy_scale / jerk_scale 权重

无论哪种结果，都是有价值的实证贡献。

    两条互补路径¶

    路径 1 (第三阶段): 提升环境保真度
  改进 A --> Simulink Fossen 模型
  智能体通过经验隐式学习物理
  代价: 重、慢 (~50 步/秒)

                              --> 第四阶段: A + B + C 全包含
路径 2 (第二阶段): 提升智能体物理感知
  改进 B + C --> 在轻量环境中做 PIRL
  智能体通过观测和奖励显式学习物理
  代价: 轻、快 (~2000 步/秒)

    第四阶段的关键实验：当 A 已经是高保真时，B+C 是否还能提供额外价值？如果能，说明显式物理感知比隐式经验学习更有效。

    学术定位

    PIRL 概念源自 PINN (Physics-Informed Neural Networks)，但在 RL 领域尚无统一范式。USV 导航中的 PIRL 研究几乎空白。本项目聚焦位置 B+C 这条低风险、高可行的路径，为 USV PIRL 提供首个实证基准。
