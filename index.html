C4 Competition Background

The National Marine Vehicle Design and Construction Competition is the highest-level, largest-scale national competition in China's shipbuilding and ocean engineering domain. Guided by the China Association for Science and Technology and the Ministry of Industry and Information Technology, it promotes industry-academia integration and cultivates high-level engineering talent.

C4 Intelligent Navigation Track

Task

Train a USV using deep reinforcement learning to autonomously navigate around obstacles and reach a designated target point in the official Unity simulation platform.

Official Platform

Unity + ML-Agents simulation environment provided by the organizing committee. Our project builds a lightweight Python/Gym equivalent for rapid algorithm prototyping.

Scoring

Objective metrics: success rate, completion time, and path length. Subjective review: algorithm novelty, technical report quality, and oral defense.

Timeline (2025)

May 16-31: Registration. Jun 1 - Jul 21: Regional competition. Jul 31: Regional review. Aug 8-11: National finals at Jiangsu JUST.

How this project relates to C4: We developed ID3QN as a candidate algorithm for the C4 track. While the official competition uses Unity/ML-Agents, our Python/Gym implementation faithfully replicates the core task (USV obstacle avoidance + goal reaching) for faster iteration. The algorithm design, training pipeline, and analysis methodology are directly applicable to the competition.

Project Overview

This project proposes an Improved Dueling Double DQN (ID3QN) algorithm for unmanned surface vehicle (USV) autonomous navigation in the C4 Intelligent Ship Competition scenario. ID3QN integrates prioritized experience replay, composite reward shaping, soft target updates, and learning rate scheduling, achieving 83% success rate in a randomized obstacle environment.

Simulation

20m x 20m water area, 3-5 random obstacles, 180-ray LiDAR, 184-dim state space, 5 discrete actions

Algorithms

4 progressive variants: DQN -> DDQN -> D3QN -> ID3QN, enabling systematic comparison

Training

2000 episodes with GitHub Actions CI/CD, fully reproducible with configurable seeds

Paper

IEEE Transactions format, 5 sections, 8 figures, 14 tables, 20 references

ID3QN Algorithm

Key Improvements over D3QN

Prioritized Replay

SumTree-based PER with O(log N) sampling. High TD-error transitions replayed more frequently. IS weight annealing (beta: 0.4->1.0).

Reward Shaping

3-component dense reward: distance progress + heading alignment + safety penalty. Transforms sparse C4 signals into continuous guidance.

Soft Update

Polyak averaging (tau=0.005) replaces hard copy. Smoother target network transitions reduce value oscillation.

LR Scheduling

StepLR decay: halve every 500 episodes. Fast exploration early, fine-tuning later. Huber loss + gradient clipping.
