# 仿真环境

c4nav-core 仿真环境的设计：从竞赛规则到 MDP 建模。

## 背景：什么是 MDP？

**MDP（马尔可夫决策过程）** 是强化学习的数学基础框架，用五元组描述一个序贯决策问题：

$$MDP = (S, A, T, R, \gamma)$$

| 符号 | 含义 | 通俗解释 |
|------|------|---------|
| **S** | 状态空间 | 世界的所有可能状态 |
| **A** | 动作空间 | 智能体能做什么 |
| **T(s,a,s')** | 转移函数 | 执行动作后世界怎么变化 |
| **R(s,a)** | 奖励函数 | 什么结果是好的 |
| **γ** | 折扣因子 | 多看重未来奖励（0~1 之间） |

智能体在每个时间步观察状态、选择动作、获得奖励、进入新状态，目标是找到一个**策略**使累计奖励最大化：

```
状态 s0 --> 动作 a0 --> 奖励 r0 --> 状态 s1 --> 动作 a1 --> 奖励 r1 --> ...

目标：最大化 r0 + γr1 + γ²r2 + γ³r3 + ...
```

!!! info "马尔可夫性质"
    "下一个状态只取决于当前状态和动作，与历史无关。"即：知道船**现在**的位置、速度、周围障碍物就够了，不需要知道 10 步之前在哪。这个假设简化了问题，也是 DQN 等算法能工作的前提。

## 设计原则：竞赛规则 → MDP 编码

C4 竞赛规则不是直接写进算法的（不是 if-else 硬编码避障逻辑），而是通过 **MDP 建模**间接编码到 RL 框架的要素中。规则定义了问题的边界条件，算法自己找解法：

| 竞赛规则 | MDP 要素 | 代码模块 | 智能体视角 |
|---------|---------|---------|-----------|
| 水域规格（20m x 20m，3-5 个障碍物） | 初始状态分布 S₀ | `scene.py` | "每局开始时世界长什么样" |
| 船体参数（航速 1.8m/s，转向率 45°/s） | 状态转移函数 T | `usv_dynamics.py` | "我执行动作后世界怎么变" |
| 评分规则（到达得分，碰撞扣分） | 奖励函数 R | `reward.py` | "什么结果是好的" |
| 控制方式（舵角 + 油门） | 动作空间 A | `usv_env.py` 中的 `ACTION_MAP` | "我能做什么" |
| 传感器能力（LiDAR 量程 18m） | 观测空间 O | `lidar_sensor.py` + `usv_env.py` | "我能看到什么" |

!!! example "对比理解"
    **规则系统**（传统做法）：人告诉机器"怎么做" -- `if 快撞了 then 转向`

    **MDP 编码**（RL 做法）：人告诉机器"世界是什么样的、什么结果是好的" -- 碰撞 = -100 分，智能体在训练中反复碰壁，**自己学会**了避障

c4nav-core 中的 γ = 0.99，意味着智能体很看重长远收益，不会只贪图眼前的小奖励而忽视未来的碰撞风险。
